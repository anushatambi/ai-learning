🤖 Agent Breakdown (8 Agents Total)

🎛️ Orchestrator
No LLM · Python
Master coordinator: spawns agents, phase sequencing, error handling, memory read/write.
🔍 Research Subagent
🧠 Phi-3 mini (Ollama)
3 search iterations, DuckDuckGo, RSS, scraping → 8-15 raw candidates.
⭐ Quality & Curation
🧠 Llama 3.2 3B (Ollama)
Scores recency, authority, impact, novelty, relevance → Top 6 + HIGH IMPACT flag.
👤 User Proxy (HITL)
No LLM · CLI
Displays top6 per section → user selects 4 → returns 20 confirmed items.
✍️ Content Synthesis
🧠 Llama 3.2 3B
Fetches article → 50-word faithful summary + cleaned headline.
🖼️ Media Generator
Tool-based + Diffusers
OG image → Unsplash → Stable Diffusion (local) → 16:9 resize.
📦 Rendering Agent
No LLM · Jinja2+pdfkit
11 quality checks → HTML email + PDF + metadata.json.
🧠 Memory Agent
Embedded in Orch
Deduplication log, last 4 issues, story hashes.

⏱️ Agent Interaction Sequence

Orchestrator → Section Agents (5 parallel) → 3 iterations per section (Research Subagent + DuckDuckGo + RSS) → Quality Agent scores each candidate (5 factors) → returns Top 6 per section Orchestrator → User Proxy Agent → CLI displays Top6 × 5 sections → User selects 4 per section → returns confirmed 20 items Orchestrator → Content Agents (20 parallel) → fetch article + Llama 3.2 summary (50 words ±3) Orchestrator → Media Agents (20 parallel) → OG image → Unsplash → Stable Diffusion fallback → save 16:9 800x450 JPG Orchestrator → Rendering Agent → 11 quality checks → Render HTML email + PDF + metadata.json → Write to /output/YYYY-MM-DD/

📰 5 Newsletter Sections (4 items each → 20 total)

  • Section 1: AI & ML: Top Developments (models, papers, benchmarks)
  • Section 2: Dev & Open Source (GitHub, frameworks, libraries)
  • Section 3: Industry News (products, funding, regulation, hiring)
  • Section 4: Hackathon & Competition (Kaggle, Devpost, fellowships)
  • Section 5: Data Science & Analytics (BI, MLOps, data infra)

✅ 11 Pre-Publish Quality Checks (Rendering Agent)

1. All 20 items present
2. Summaries within 47–53 words
3. All links reachable (HTTP 200)
4. No duplicate URLs (hash)
5. No items >14 days old
6. All images load
7. Max 2 same source/section
8. Headlines ≤120 chars
9. Alt text on all images
10. metadata.json generated
11. HTML email-safe + responsive

📁 Output Directory Structure

/output/YYYY-MM-DD/ ├── newsletter.html # Email-ready, inline CSS ├── newsletter.pdf # PDF snapshot ├── metadata.json # Full run metadata + quality report └── images/ ├── sec1_item1.jpg ├── sec1_item2.jpg └── ... (20 files, 16:9, 800x450)

🧠 Memory & Deduplication

Within issue: Priority assignment (Section 1 > 3 > 2 > 5 > 4) · story never repeats

Across issues: Maintain log of last 4 issues (URLs + story hashes) · 30-day memory

Exceptions: Major ongoing stories allowed with new developments
